查看工作目录当前状态：git status
On branch master
Your branch is up-to-date with 'origin/master'.


Untracked files:
(use "git add <file>..." to include in what will be committed)


        test-list.txt


nothing added to commit but untracked files present (use "git add" to track)
以上内容包含信息：
1.你在 master branch
2.当前 branch 没有落后于 origin/master
3.你有 untracked files （未追踪的文件），文件名是 test-list.txt
4.你可以使用 git add 来开始追踪文件
每个文件有“changed/unstaged”（已修改），"staged"（已修改并暂存），"commited"（已提交）三种状态，以及一种特殊状态"untracked"（未跟踪）。
test-list.txt当前属于“untracked”状态，需要先 add 来让Git开始跟踪它：git add test-list.txt，
此时不会有反馈信息，再一次 git status 
On branch master
Your branch is up-to-date with 'origin/master'.

changes to be committed:
 (use "git reset HEAD <file>..." to unstage)
            new file:        test-list.txt
此时 test-list.txt 这个文件从"untracked"(未跟踪）变成了"staged"(已暂存），即该文件中被改动的部分被记录进了 staging area （暂存区）。
stage：集中收集改动以待提交
所谓的 staging area ，是 .git 目录下一个叫做 index 的文件
提交指令：git commit    （ commit 之前先看看 status 是个好习惯）
小写i切换到插入模式，Esc返回命令模式，然后连续输入两个大写的“Z”进行保存。
[master 79c353d] Add "test-list.txt"
1 file changed, 10 insertions(+)
create mode 100644 test-list.txt
把提交推送到中央仓库：git push
先 git status 查询状态
On branch master
Your branch is ahead of 'origin/master' by 2 commits.
 (use "git push" to publish your local commits)

nothing to commit, working tree clean
当前 branch 已经领先于("ahead of") 'origin/master' 两个提交了。origin/master 中的 origin 是远端仓库的名称，是你在用 clone 指令初始化本地仓库时 Git 自动帮你起的默认名称；master 是 origin 上的分支名。可以暂时把 origin/master 简单理解为中央仓库。
然后按照提示 git push。由于这是联网操作，所以在这个过程 GitHub 会再次向你索要账户和密码。

Caching your GitHub password in Git： https://help.github.com/en/articles/caching-your-github-password-in-git

把同事 push 的新代码取下来：git pull
